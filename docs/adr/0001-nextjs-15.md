# ADR 0001 — Adopt Next.js 15 (App Router)

- **Status:** Accepted
- **Date:** 2026-05-11
- **Deciders:** Engineering lead

## Context

The Lovable prototype is built on TanStack Start + Vite. The SOP §2.1 mandates **Next.js 15 App Router** for the production rebuild.

## Decision

Migrate from TanStack Start to Next.js 15 App Router with React 19 Server Components, file-based routing, SSG for the marketing page, and Node runtime API routes for forms.

## Consequences

**+** Native ISR/SSG, built-in `next/image` + `next/font`, generous Vercel free tier, edge-runtime support, OG image generation via `next/og`, native sitemap/robots/manifest helpers.
**+** Industry-standard hire-ability — most full-stack React engineers know Next.
**−** GSAP must be wrapped in `"use client"` and `useGSAP({ scope })` (mild ceremony).
**−** Tailwind 4 with App Router requires the `@theme inline` + `@import "tailwindcss"` pattern in `globals.css` (no longer a `tailwind.config.ts`).

## Alternatives considered

- **Keep TanStack Start** — rejected: SOP explicitly mandates Next.js; smaller ecosystem.
- **Remix** — rejected: not in SOP; weaker static-export story.
- **Astro** — rejected: section animations + React-only API made it more friction than benefit.
